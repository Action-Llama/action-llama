# E2E Coverage Gaps

Priority list of untested or under-tested e2e flows. The `e2e-coverage-improver`
agent reads this file and works top-down. Mark items `[x]` when a test exists.

## High Priority

- [x] Webhook trigger handling ‚Äî deliver a mock webhook payload and verify agent execution (cli-flows.test.ts has a `todo` placeholder)
- [x] Scheduler cron triggering ‚Äî verify agents fire on schedule and not before
- [x] Error: invalid credentials ‚Äî start scheduler with bad/missing credentials, verify graceful error
- [x] Error: Docker unavailable ‚Äî start scheduler when Docker CLI fails, verify error reporting
- [x] Error: SSH connection failure during VPS deploy ‚Äî simulate SSH timeout, verify rollback behavior
- [x] Multi-agent scheduling ‚Äî configure 3+ agents with different schedules, verify independent execution
- [x] Agent timeout enforcement ‚Äî configure short timeout, verify agent is killed after expiry

## Medium Priority

- [x] Agent concurrent scaling ‚Äî set `scale = 2`, trigger two events, verify parallel execution
- [x] Webhook event filtering ‚Äî deliver events that don't match labels/actions/branches, verify they are ignored
- [x] Scheduler pause/resume with in-flight agents ‚Äî pause while agent is running, verify it completes
- [x] MCP server integration ‚Äî start scheduler with MCP config, verify MCP tools are available to agents
- [x] Extension loading ‚Äî configure an extension, verify it loads and registers
- [x] Dashboard SSE reconnection ‚Äî disconnect SSE stream, verify auto-reconnect and data continuity
- [x] Credential rotation ‚Äî update credentials while scheduler is running, verify new creds are used

## Lower Priority

- [x] Agent-to-agent communication ‚Äî one agent triggers another via `al-rerun`
- [x] Log streaming ‚Äî verify `al logs` streams live output from running agents
- [x] Deployment with custom Dockerfile ‚Äî push agent with custom Dockerfile, verify image build
- [x] Deployment upgrade path ‚Äî deploy v1, upgrade to v2, verify agents restart with new code
- [x] Resource lock contention ‚Äî two agents competing for the same lock, verify mutual exclusion
- [x] Webhook retry/deduplication ‚Äî deliver duplicate webhook, verify single execution
- [x] Stats and usage tracking ‚Äî run agents, verify stats API returns correct execution counts
